# منظومة حلقات تحفيظ القرآن الكريم — مسجد الغفران

منظومة إلكترونية مجانية ومفتوحة المصدر لإدارة حلقات تحفيظ القرآن الكريم: تسجيل الطلبة، الحضور والغياب، متابعة الحفظ والمراجعة مربوطة بالسورة ورقم الآية، بصلاحيتين (الشيخ الرئيسي / المدرس)، مع تقارير قابلة للطباعة والتصدير كـ PDF.

## التقنيات المستخدمة

- **Next.js 16** (App Router) + TypeScript
- **Drizzle ORM** + SQLite (محلياً) / **Turso** (في الإنتاج)
- **Auth.js (NextAuth v5)** لتسجيل الدخول والصلاحيات
- **Tailwind CSS v4** بواجهة عربية RTL كاملة
- تصدير PDF عبر الطباعة المباشرة من المتصفح (بدون مكتبات إضافية)

## التشغيل محلياً

**المتطلبات**: [Node.js](https://nodejs.org) نسخة 20.9 أو أحدث.

```bash
npm install
cp .env.example .env
# ولّد سر الجلسات وضعه في AUTH_SECRET داخل .env:
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

npm run db:migrate   # إنشاء قاعدة البيانات المحلية
npm run db:seed      # إنشاء حساب الشيخ الرئيسي الافتراضي
npm run dev          # تشغيل الخادم على http://localhost:3000
```

بيانات الدخول الافتراضية (يمكن تغييرها في `.env` قبل تشغيل `db:seed`، ويُنصح بتغيير كلمة المرور من صفحة "حسابي" بعد أول دخول):

- البريد الإلكتروني: `admin@ghofran.local`
- كلمة المرور: `ChangeMe123!`

## النشر المجاني (Vercel + Turso)

قاعدة البيانات المحلية عبارة عن ملف SQLite، وهو غير مناسب لاستضافة Vercel المجانية (بيئة بدون تخزين دائم). لذلك نستخدم **Turso**، وهي قاعدة بيانات SQLite سحابية مجانية بنفس المحرك تماماً:

1. أنشئ حساباً مجانياً على [turso.tech](https://turso.tech) وأنشئ قاعدة بيانات جديدة.
2. احصل على `Database URL` و `Auth Token` من لوحة تحكم Turso.
3. طبّق الهجرات على القاعدة السحابية:
   ```bash
   TURSO_DATABASE_URL=... TURSO_AUTH_TOKEN=... npm run db:migrate
   TURSO_DATABASE_URL=... TURSO_AUTH_TOKEN=... npm run db:seed
   ```
4. في مشروع Vercel، أضف متغيرات البيئة التالية: `TURSO_DATABASE_URL`، `TURSO_AUTH_TOKEN`، `AUTH_SECRET`.
5. اربط المستودع بـ Vercel وانشر المشروع.

## هيكل المشروع

```
app/(app)/        صفحات النظام بعد تسجيل الدخول (لوحة التحكم، الطلبة، الحضور، الحفظ، التقارير...)
app/login/        صفحة تسجيل الدخول
lib/db/           مخطط قاعدة البيانات (Drizzle) والاتصال بها
lib/auth/         إعداد Auth.js وطبقة الصلاحيات المركزية
lib/actions/      إجراءات الخادم (Server Actions) لكل وحدة
lib/quran/        بيانات السور الثابتة (114 سورة) ودوال التحقق من مدى الآيات
components/       مكونات الواجهة (نماذج، جداول، عناصر UI أساسية)
```

## الصلاحيات

- **الشيخ الرئيسي**: يضيف/يعدّل/يحذف المدرسين والحلقات، ويطّلع على كل الطلبة والتقارير.
- **المدرس**: يدير طلبة حلقته فقط (يُفرض هذا النطاق على مستوى الخادم في كل الإجراءات، وليس فقط في الواجهة).

## أوامر مفيدة

| الأمر | الوصف |
|---|---|
| `npm run dev` | تشغيل خادم التطوير |
| `npm run build` | بناء نسخة الإنتاج |
| `npm run db:generate` | توليد ملف هجرة جديد بعد تعديل `lib/db/schema.ts` |
| `npm run db:migrate` | تطبيق الهجرات على قاعدة البيانات |
| `npm run db:seed` | إنشاء حساب الشيخ الرئيسي الافتراضي |
| `npm run db:studio` | فتح واجهة Drizzle Studio لتصفح قاعدة البيانات |
